# Documentation

This directory contains the project definition for ResearchGit before implementation is built out.

## Current Focus

The current priority is to define a realistic v1 for a Git-native academic writing workflow. That means narrowing the document model, citation model, and collaboration assumptions before implementation starts.

## Structure

- `product/`: problem statement, vision, and scope
- `architecture/`: system-level design and component boundaries
- `workflows/`: key user and document lifecycle flows
- `roadmap.md`: staged delivery plan

## Recommended Reading Order

1. `product/vision.md`
2. `product/v1-scope.md`
3. `workflows/manuscript-lifecycle.md`
4. `architecture/overview.md`
5. `roadmap.md`

## Open Decisions

- how opinionated the project layout should be
- how citations should be represented internally beyond external formats
- what the first remote collaboration model should look like
- how much export automation belongs in v1
