# Roadmap

## Phase 0: Definition

- establish repository structure
- document product scope and non-goals
- decide the canonical text-first manuscript workflow
- define manuscript and citation workflow assumptions
- prepare architecture notes for implementation

## Phase 1: Document And Citation Model

- initialize a ResearchGit project
- support `LaTeX`, `Markdown`, and `Typst` as first-class text formats
- define tracked manuscript assets
- define citation asset structure
- record revision metadata and history

## Phase 2: Collaboration Workflow

- introduce branch and merge concepts for manuscripts
- support author attribution and reviewable changes
- define review and revision-round assumptions
- define remote synchronization assumptions

## Phase 3: Export And Citation Integration

- support BibTeX-compatible workflows
- validate citation usage against manuscript state
- improve bibliography update handling

## Phase 4: Automation Layer

- add document quality checks
- add structured reporting workflows
- explore AI-assisted literature and writing support

## Phase 5: Submission Pipeline

- support export and packaging for submission
- support revision-round comparisons
- support journal or conference workflow extensions
